重要提示：

  • Token 必须有完整的 repo 权限才能创建 Release
  • 文件将存储在 GitHub Releases，支持大文件上传
  • 通过后端代理服务上传，避免 CORS 跨域问题
  • Token 只保存在浏览器本地，非常安全

遇到问题？

  • 403 Forbidden: Token 权限不足，请确保已勾选完整 repo 权限
  • 422 Unprocessable: 需要先在仓库页面手动创建一个 Release
  • 404 Not Found: 检查仓库名和用户名是否正确

使用说明

Fork 本仓库并部署 docs 到 GitHub Pages 后，在本页填好 Token、仓库所有者、仓库名、Release 标签（如 latest），保存即可直接上传/下载，无需先手动创建 Release，后端会自动创建。

常见问题

文件存在哪？

存在 GitHub Releases，服务器只做中转，不落盘。

支持多大文件？

单文件最大 2GB。

Token 安全吗？

只存在浏览器本地，后端不存。

422 或 403 怎么办？

422：检查 Token 是否勾选完整 repo 权限。403：若用的是别人的 Pages 地址，需 Fork 后部署自己的 Pages 再填信息使用。
