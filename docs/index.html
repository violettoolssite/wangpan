📖 使用指南

1

创建 GitHub Token（重要）

访问 GitHub Settings → Tokens
⚠️ 必须勾选 repo 权限（包括 repo:status, repo_deployment, public_repo, repo:invite, security_events）

2

创建存储仓库

在 GitHub 创建一个新仓库： https://github.com/new
可以是 public 或 private，记录下仓库名

3

连接并使用

填写 Token 和仓库信息，点击"连接仓库"，开始上传文件

常见错误解决

403 Forbidden - 创建 Release 失败

这是最常见的问题。解决办法：
1. 确保生成 Token 时勾选了完整的 repo 权限
2. 如果仓库是私有的，确保你有写权限
3. 尝试在 GitHub 仓库页面手动创建一个 Release

404 Not Found - 仓库未找到

检查以下几点：
1. 仓库所有者（用户名）是否正确
2. 仓库名拼写是否正确
3. 如果是私有仓库，请确保 Token 可以访问

文件能上传但无法删除

删除权限可能不足，请检查 Token 是否有完整的 repo 权限

文件存储在哪里？

文件存储在 GitHub Releases，这是 GitHub 官方的文件分发功能

有存储限制吗？

单个文件最大约 100MB（GitHub API 限制），仓库总大小建议不超过 1GB

安全吗？

完全安全。Token 只保存在你的浏览器本地，直接调用 GitHub API，不经过任何中间服务器

可以使用私有仓库吗？

可以，你的 Token 需要有访问该私有仓库的完整权限
