遇到问题？

  • 403 Forbidden: Token 权限不足，请确保已勾选完整 repo 权限

  • 404 Not Found: 检查仓库名和用户名是否正确

使用指南

1

在仓库创建一个 Release

访问 GitHub Releases/New
输入 Tag: v1.0.0，创建一个初始 Release

2

创建 GitHub Token（完整权限）

访问 GitHub Settings → Tokens
必须勾选 repo 及其所有子权限

3

连接并使用

填写 Token 和仓库信息（Tag 使用刚创建的），开始上传文件

常见问题

422 错误 - 创建 Release 失败

这是最常见的问题。解决办法：
1. 确保生成 Token 时勾选了完整的 repo 权限
2. 在仓库页面手动创建一个 Release
3. 不要使用不存在的 TAG 名称

文件存储在哪里？

文件存储在 GitHub Releases，这是 GitHub 官方的文件分发功能，支持大文件版本管理

支持多大的文件？

单个文件最大 2GB，仓库总大小建议不超过 1GB

安全吗？

完全安全。Token 只保存在你的浏览器本地，通过后端代理与 GitHub API 通信

可以使用私有仓库吗？

可以，你的 Token 需要有访问该私有仓库的完整权限
